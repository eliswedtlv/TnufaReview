# TnufaReview — STATUS

**Last updated:** 2026-07-12 — **LIVE at https://tnufareview.com/**, reviews working end-to-end (~under 1 min); favicon + OG link-preview added (T-1141); title font Rubik (T-1142)

## What it is
Web tool that reviews Israel Innovation Authority "Tnufa" grant applications. Founder uploads a Word (.docx) file → backend extracts the text → AI reviews each of the 11 Tnufa sections against Tnufa expectations → returns structured Hebrew comments per section.

## Now / current state
- **Backend** `app.py` (Flask) on OpenRouter (`deepseek/deepseek-v4-pro`). Accepts **any readable `.docx`** (Tnufa gate removed). Reviewer is now **v2** (T-1137): the prompt + per-section criteria live in `review_system_prompt.txt` + `instructions_form.json` (loaded once at startup, fail-fast if missing). Extraction is **AI-only** (T-1138): the deterministic `extract_sections_structural` path was removed because real filled forms are saved without heading styles, so it never mapped cleanly and `/review` always used the AI path anyway. Flow is now `extract_all_text_from_docx` (paragraphs + table cells, incl. the exec-summary answer inside a table cell) → `ai_extract_sections` → single review call (2 LLM calls total: extraction + review). Review is a **single** OpenRouter call over all 11 sections with `instructions_form` embedded. Section field is `applicant_answer`. Env var resolves as `OPENROUTER_API_KEY` with legacy `OPENEOPUTER_API_KEY` fallback. **Live model = `google/gemini-3.1-flash-lite`** (set via `OPEN_ROUTER_MODEL` on Railway); the code default is still `deepseek/deepseek-v4-pro`. **Model note:** DeepSeek V4 Pro is a reasoning model and was far too slow here (~4.7 min on extraction alone → ~8–10 min total → Railway 502); Gemini 3.1 Flash Lite (non-reasoning) brings a full review to ~under 1 min.
- **Single service:** Flask serves `index.html` at `/`, `/health` returns `{"status":"ok"}`, `/review` unchanged. `Procfile` binds gunicorn to `0.0.0.0:$PORT` with 2 workers, a **600s** worker timeout, and `--access-logfile -/--error-logfile -` (T-1140) so Railway can reach it, long multi-minute reviews aren't killed, and gunicorn access logs reach stdout.
- **Observability** (T-1140): `/review` logs (stdout, so it shows in Railway) request received → extraction start/done (elapsed + extracted chars) → review-call start/done (elapsed) → total elapsed, and every `except` block logs `traceback.format_exc()` before returning the JSON error. Both OpenRouter calls (extraction + review) pass `timeout=240` so a hung upstream fails fast with a clean JSON error instead of hanging until the worker is killed.
- **Frontend** single `index.html`, Hebrew RTL — **redesigned** (T-1133): warm-paper/evergreen visual system, Rubik (title) + Assistant (body) fonts, accessible upload dropzone, staggered load. Tnufa-only messaging/modal removed (accepts any Word `.docx`). Backend call is now the **relative** `/review` — **requires the single-service deploy** (Flask serving the page); it will NOT work on GitHub Pages. `prettifyJSON` 11-section keys/titles/order preserved unchanged.
- **Hosting:** **live at https://tnufareview.com/** — single Railway service (also `web-production-17375.up.railway.app`). The old GitHub Pages URL is stale/broken and should be retired.
- **Branding/link preview** (T-1141): on-brand `favicon.svg` (evergreen rounded tile + white "T"), `favicon.ico` fallback, 180×180 `apple-touch-icon.png`, and a 1200×630 `og-image.png` (warm-paper card, RTL Hebrew slogan). `<head>` has the icon links + Open Graph/Twitter tags (`og:image`/`twitter:image` = absolute `https://tnufareview.com/og-image.png`). Flask serves all four assets from `app.root_path` via `send_file`, which also fixes the `GET /favicon.ico → 404`.
- Tests in `tests/` (pytest, stubbed LLM) — 9 passing. `pytest` is a dev-only dep, **not** in `requirements.txt`.
